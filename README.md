---

## 这是什么？（3句话说明白）
我们在做一件事：把孩子的成长过程，用**研究化 + 项目化**的方法推进——  
- 不靠鸡汤，不靠临时抱佛脚  
- 用可执行计划、可记录数据、可复盘迭代的方式持续改进  
- 让孩子获得可迁移能力：学习能力、创造能力、表达能力、AI协作能力

## 我们的 AI 协作模式（为什么有仓库 + GPTS + Claude Projects）
这个仓库主要负责 **存放事实与产出**：计划、日志、复盘、项目材料、指标定义等。  
我们把它当作唯一事实源（SSOT）。

"思考与推理"主要由两套 AI 协作系统完成：
- **GPTS（OpenAI）**：负责把目标拆成可执行任务与模板，输出标准化的"结论 / 下一步最小动作 / 需要输入 / 入库稿"，推动更新仓库。
- **Claude Projects（Anthropic）**：作为另一套推理与写作引擎，用于深度修订文档、生成方案、结构化复盘（同样以仓库为唯一事实源）。

无论使用哪套 AI，规则一致：  
**AI 只能在"读取仓库内容（或你粘贴的片段）"基础上推理；最终结论必须以 Markdown 形式沉淀回仓库，方便协作与复盘。**

**边界说明**：仓库负责"可追踪的证据与产出"；AI 负责"推理与生成"；人负责"选择与执行"。

## 工作流（最简版）
1) 仓库提供现状：INDEX / 上周复盘 / 数据日志  
2) GPTS 或 Claude Projects 生成：本周计划 / 任务拆解 / 模板  
3) 研究团队执行并记录：日打卡、材料整理、项目推进  
4) 周复盘：数据 → 洞察 → 调整  
5) 更新仓库（SSOT）：INDEX、计划、复盘、项目文件

## 协作分工建议（两人团队）
- **研究负责人**：方案、复盘、指标体系、阶段报告结构、关键沟通
- **执行负责人**：日常推进、打卡监督、资料整理、材料产出与归档

## 我们如何衡量进展（不追求完美，但追求可验证）
我们关心五条主线的"可观察变化"：
1) 身体发育与运动
2) AI 素养与工具能力
3) 学业与基础能力
4) 创新与科创项目
5) 表达与传播

每周至少发生一次：
- 有数据的记录
- 有产出的推进
- 有复盘的调整

# yangtongxue｜立体教育研究仓库（SSOT）

> 这是一个"家庭教育 × 科创 × 表达 × AI素养"的长期研究型项目仓库。  
> **本仓库是唯一事实源（Single Source of Truth, SSOT）**：计划、日志、复盘、项目材料都以这里为准。

---

## 这是什么？（3句话说明白）
我们在做一件事：把孩子的成长过程，用**研究化 + 项目化**的方法推进——  
- 不靠鸡汤，不靠临时抱佛脚  
- 用可执行计划、可记录数据、可复盘迭代的方式持续改进  
- 让孩子获得可迁移能力：学习能力、创造能力、表达能力、AI协作能力

## AI 协作架构图（文字版）

### GitHub 仓库
- **角色**：资料与证据库
- **作用**：存放所有计划、日志、复盘、项目材料、指标定义
- **定位**：唯一事实源（SSOT），所有决策和产出都以这里为准

### GPTS（OpenAI）
- **角色**：日常教练与生成
- **作用**：把目标拆成可执行任务，生成标准化模板和文档
- **输出格式**：结论 / 下一步最小动作 / 需要输入 / 入库稿
- **使用场景**：周计划生成、任务拆解、日常问答、文档模板生成

### Claude Projects（Anthropic）
- **角色**：深度研究/资料梳理
- **作用**：深度分析、方案设计、结构化复盘、研究报告
- **使用场景**：复杂问题分析、长期规划、阶段报告、深度文档修订

**协作原则**：AI 只能在"读取仓库内容（或你粘贴的片段）"基础上推理；最终结论必须以 Markdown 形式沉淀回仓库，方便协作与复盘。

**边界说明**：仓库负责"可追踪的证据与产出"；AI 负责"推理与生成"；人负责"选择与执行"。

👉 详细说明见：[`docs/AI协作模式说明.md`](docs/AI协作模式说明.md)

## 工作流（最简版）
1) 仓库提供现状：INDEX / 上周复盘 / 数据日志  
2) GPTS 或 Claude Projects 生成：本周计划 / 任务拆解 / 模板  
3) 研究团队执行并记录：日打卡、材料整理、项目推进  
4) 周复盘：数据 → 洞察 → 调整  
5) 更新仓库（SSOT）：INDEX、计划、复盘、项目文件

## 隐私与边界（简版）

### 脱敏要求
- 任何对外可展示材料必须脱敏：不出现真实姓名、学校、住址、可识别照片等
- 本仓库默认只存"可研究/可复盘"的信息，不存敏感个人信息
- `06_reports/` 目录下的所有材料必须完全脱敏

### 健康/心理边界
- 我们只做**观察与习惯建议**，不做诊断
- 必要时建议专业咨询
- 不记录可能涉及隐私的健康细节

### 什么不入库
- 未脱敏的个人信息
- 健康诊断记录（只记录观察和习惯建议）
- 家庭内部敏感对话
- 涉及隐私的心理评估细节

👉 详细规则见：[`00_admin/OPS.md`](00_admin/OPS.md) 和 [`docs/AI协作模式说明.md`](docs/AI协作模式说明.md)

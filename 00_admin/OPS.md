# 运行手册（OPS）

## 项目是什么
这是一个"研究化 + 项目化"的家庭教育长期项目。我们通过可执行计划、数据记录、复盘迭代的方式，培养孩子的学习能力、创造能力、表达能力、AI协作能力。

## SSOT 规则（仓库唯一事实源）
- **所有计划、日志、复盘、项目材料都以本仓库为准**
- 任何决策、调整、数据更新必须同步到仓库
- 不要依赖口头沟通或外部文档，一切以 Git 提交为准

## 每周节奏（MVP 周节奏）

### 周一：周计划
- 生成周计划文档：`02_plans/YYYY-Www-plan.md`
- 明确本周 MVP（最小可行版本）目标
- 分配 Owner 和 Due Date
- 更新 `00_admin/INDEX.md` 的"本周最小目标"

### 周三：检查点
- 检查进度，识别阻塞
- 必要时调整计划（记录在日志中）

### 周日：周复盘
- 生成周复盘文档：`03_logs/YYYY-Www-review.md`
- 数据 → 洞察 → 调整
- 更新 `00_admin/INDEX.md` 的"最近四周复盘"
- 为下周计划做准备

## 即时反馈系统执行流程

### 每次课开头（5分钟热身）

**Ops Lead必须完成**：
1. 舒尔特表训练（5分钟）
   - 作为"开机仪式"，帮助孩子进入专注状态
   - 记录完成时间
   - 详细操作见：`docs/舒尔特表训练指南.md`

### 每次课后（10分钟内）

**Ops Lead必须完成**：
1. 记录课后4行（用时、做了什么、结果、下一步）
2. 粘贴到GPTS
3. GPTS自动生成即时反馈卡
4. 发给孩子看（通过微信/打印）
5. 更新 `00_admin/ability-levels.md`

**反馈卡包含**：
- 五维能力经验值变化
- 升级状态
- 解锁权限
- 距离下一个里程碑
- 下次小目标

**模板**：`docs/模板_课后即时反馈卡.md`

### 每周日（周复盘时）

**Research Lead必须完成**：
1. 读取本周所有反馈卡
2. 生成能力雷达图
3. 生成本周成长报告
4. 更新 `00_admin/ability-levels.md`

**目的**：
- 家长可视化孩子成长
- 识别薄弱环节
- 调整下周计划

## 最低交付物清单

| 交付物 | 路径 | 命名规范 | 频率 |
|--------|------|----------|------|
| 周计划 | `02_plans/` | `YYYY-Www-plan.md` | 每周一 |
| 周复盘 | `03_logs/` | `YYYY-Www-review.md` | 每周日 |

- 周计划：`02_plans/2025-W01-plan.md`
- 周复盘：`03_logs/2025-W01-review.md`
- 日打卡：`03_logs/2025-12-21-daily.md`
- 会议纪要：`03_logs/2025-12-21-meeting-notes.md`
- 项目文件夹：`04_projects/project-xxx/`

## 角色分工

| 角色 | 职责 |
|------|------|
| **家长** | 方向把关、价值判断、资源支持、最终决策 |
| **研究负责人** | 方案设计、复盘分析、指标体系、阶段报告结构、关键沟通 |
| **执行负责人** | 日常推进、打卡监督、资料整理、材料产出与归档 |
| **孩子** | 项目主导、执行任务、记录过程、参与复盘 |

## 更新 INDEX.md 的规则
- 每次周计划后，更新"本周最小目标（MVP）"
- 每次周复盘后，更新"最近四周复盘"
- 项目启动/完成时，更新"正在进行的项目"
- 保持 INDEX 反映当前真实状态，不要滞后

## 隐私与边界

### 脱敏要求
- 任何对外可展示材料必须脱敏：不出现真实姓名、学校、住址、可识别照片等
- 本仓库默认只存"可研究/可复盘"的信息，不存敏感个人信息
- `06_reports/` 目录下的所有材料必须完全脱敏

### 健康/心理边界
- 我们只做**观察与习惯建议**，不做诊断
- 必要时建议专业咨询
- 不记录可能涉及隐私的健康细节

## 如何开始（新加入者）
1. 阅读 `00_admin/INDEX.md` 了解当前状态
2. 阅读最新周计划：`02_plans/` 目录下最新文件
3. 阅读最近 1-2 篇周复盘：`03_logs/` 目录下最新 review 文件
4. 领一个"本周最小动作"任务，直接推进

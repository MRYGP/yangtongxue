# AI 协作手册（GPTS / Claude Projects）

## 我们怎么用 AI 工具

### GPTS 与 Claude Projects 的职责分工

| 工具 | 主要职责 | 使用场景 |
|------|----------|----------|
| **GPTS** | 项目执行、日常推进、文档生成 | 周计划生成、周复盘、项目文档、日常问答 |
| **Claude Projects** | 深度分析、方案设计、研究支持 | 复杂问题分析、长期规划、研究报告 |

## 每次对话输出"四段式"

每次与 AI 协作后，输出应包含以下四段：

1. **结论**：本次对话的核心结论/产出
2. **下一步最小动作**：具体可执行的下一步（含 Owner 和 Due）
3. **需要输入**：需要人工提供的信息/决策
4. **入库稿**：可直接提交到仓库的文档/代码片段

## "别装懂"条款

**重要**：如果 AI 读不到仓库内容，必须：
- 明确声明："我无法读取仓库内容"
- 索要 3-5 个关键片段（文件名 + 关键段落）
- 基于提供的片段给出建议，不要猜测

## Prompt 模板

### A. 生成本周周计划

```
基于以下信息，生成本周周计划（格式：02_plans/YYYY-Www-plan.md）：

【上下文】
- 上周复盘要点：[粘贴上周复盘的关键洞察]
- 当前 INDEX 状态：[粘贴 INDEX.md 的当前主线]
- 本周目标方向：[简要说明]

【要求】
- 明确本周 MVP（最小可行版本）目标
- 每条任务分配 Owner 和 Due Date
- 包含五条主线的具体动作：
  1) 身体发育与运动
  2) AI 素养与工具能力
  3) 学业与基础能力
  4) 创新与科创项目
  5) 表达与传播
- 输出格式符合文件命名规范
```

### B. 生成周复盘

```
基于本周数据，生成周复盘（格式：03_logs/YYYY-Www-review.md）：

【输入】
- 本周计划：[粘贴本周计划]
- 实际执行记录：[粘贴日志/打卡数据]
- 关键事件：[简要说明]

【要求】
- 数据 → 洞察 → 调整（三段式结构）
- 明确哪些目标达成、哪些未达成及原因
- 提出下周调整建议
- 列出需要更新哪些文件（INDEX.md、计划等）
- 输出格式符合文件命名规范
```

### C. 科创项目立项

```
生成科创项目立项文档（格式：04_projects/<project_slug>/charter.md）：

【项目信息】
- 项目名称：[项目名]
- 问题描述：[要解决什么问题]
- 目标用户：[谁会用]
- 技术路径：[大概怎么做]

【要求】
- 项目 Charter（项目章程）
- 里程碑（3-5 个关键节点）
- 产物清单（可交付物）
- 时间线（预计周期）
- 风险与假设
- 输出格式符合项目文档规范
```

## 默认脱敏要求

- 所有 AI 生成的对外材料必须去标识化
- 不包含真实姓名、学校、住址、可识别照片
- 生成后人工检查一遍，确保隐私安全

## 最佳实践

1. **每次对话前**：先提供必要的上下文（INDEX、最新计划、相关日志）
2. **每次对话后**：输出"四段式"，确保有可执行的下一步
3. **文档生成后**：检查格式、命名规范、隐私脱敏
4. **定期复盘**：评估 AI 协作效果，优化 prompt 模板

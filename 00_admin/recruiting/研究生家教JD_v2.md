# 研究生研究型家教 招募说明 JD v2

## 一、我们在做什么

我们把家庭教育做成一个**可复盘的研究项目**：
- 目标 → 假设 → 训练 → 指标 → 记录 → 复盘 → 迭代
- 所有过程沉淀到GitHub作为唯一事实源（SSOT）

你不是传统家教，更像是**研究型教练 + 项目助理 + 学习方法顾问 + 文档工程师**的组合。

## 二、服务对象与基本情况

- **学生**：11岁，小学六年级，男孩
- **地点**：长沙市雨花区（线下为主，线上补充）
- **性格**：内向、腼腆，聪明有想法但表达能力弱
- **技能**：Python一级、3DOne建模，对发明创造有兴趣
- **身体**：偏矮，需要科学长高训练

（更细的个体信息与隐私资料，入选后提供并做脱敏管理）

## 四、你的核心任务（做什么）

### 1. 教学与训练执行
- 每周线下陪练与指导（学业、科创、运动、表达）
- 能把任务拆成孩子当天能完成的15-30分钟动作

### 2. 研究化闭环落地
- 把每周目标写成可验证的实验
- 定义指标并记录数据
- 每周复盘：有效/无效/调整

### 3. GitHub入库与协作
- 课后必须产出日打卡（`03_logs/YYYY-MM-DD-daily.md`）
- 每周必须产出周计划与周复盘
- 推进科创项目的产物文件

### 4. 亲子协作
- 能用低冲突方式与家长对齐优先级与节奏
- 少术语，多行动方案

## 五、你每周必须交付什么（可验收）

| 交付物 | 路径 | 频率 |
|-------|------|------|
| 周计划 | `02_plans/YYYY-Www-plan.md` | 每周一 |
| 周复盘 | `03_logs/YYYY-Www-review.md` | 每周日 |
| 日打卡 | `03_logs/YYYY-MM-DD-daily.md` | 至少4天 |
| 项目产物 | `04_projects/<slug>/` | 持续推进 |

### 验收标准

1. **周计划**必须包含：week、primary、secondary、mvp、owner、due
2. **周复盘**必须包含：week、data、insights、adjustments
3. **日打卡**至少包含：用时、做了什么、结果、下一步一小步
4. **项目产物**至少推进五件套中的一件：charter、demo_or_code_link、script、slides_outline、postmortem

## 九、AI工具说明

### 对你（老师）的定位
AI是你的**人机协同研究工具**：
- ChatGPT、Claude、Google AI Studio等
- 用于：方案设计、复盘分析、文档生成、教学辅助

## 附录C：精选案例库

### 案例1：编程教学平台（成功案例）

**背景**：某在线编程教育平台为Python初学者设计AI辅助学习系统，学员100名（年龄18-25岁），学习周期8周，目标是掌握基础Python编程能力。

**AVP测试结果**：

- $B_0$（基线）：无AI环境下完成3个编程任务，平均得分62/100
- 协作期（8周）：使用EML设计的AI助教
- W（拔线窗口）：6周完全停用AI
- $P_2$（拔线后）：完成等值任务，平均得分78/100
- **判定**：ΔC = $P_2$ - $B_0$ = 16分 > $\delta$(10分，$\approx$0.3SD) → **AVP通过**

**EML分析**：

- 有益摩擦：AI不直接给代码，而是：
- 第1-2周：提供代码框架，学员填充核心逻辑（完整性摩擦）
- 第3-4周：只给算法思路，学员自行实现（抽象性摩擦）
- 第5-6周：学员先尝试15分钟，AI才介入（延迟性摩擦）
- 维持成功率55-65%（接近目标50-70%）
- 系统性支持削减：

- 采用S型削减曲线：$S_0$=0.8 → 前2周缓慢削减 → 中期快速削减 → 后期趋于$S_{min}$=0.2
- 回退机制：2名学员因连续失败触发回退，临时提升支持后恢复

**成功原因**：

1. **摩擦与削减协同**：不是单独使用摩擦或削减，而是同时实施两者，效果互补
2. **个性化调整**：根据学员C(t)（能力向量）动态调整摩擦强度和削减速度
3. **保底机制**：$S_{min}$=0.2确保学员不会完全迷失，增强信心
4. **拔线窗口充分**：6周窗口足够让能力稳定内化

**可迁移洞察**：

- EML双条件（摩擦+削减）是**联合必要**的，缺一不可
- 50-70%成功率（工作假设）是关键平衡点：既有挑战又不挫败
- S型削减曲线优于线性：适应学习曲线的非线性特征
- 个性化调整比固定策略更有效（但实施成本更高）

**红旗提示**：不要误将“减少AI帮助"理解为“降低教学质量"。摩擦是为了促进主动学习，不是故意为难学员。如果团队抵触，先从“部分任务"或“高年级学员"试点。

### 案例2：软件团队对比实验（团队层对比）

**背景**：某科技公司的两个软件开发团队（各8人，能力相当），为期6个月的AI辅助编程实验。A组（实验组）：实施“周五无AI日"政策，B组（对照组）：无限制使用AI编程助手。

**AVP测试结果**：

- 个体层（I-AVP）：

- A组：所有8人I-AVP通过（$P_2 \geq B_0 + \delta$）
- B组：仅3人通过，5人失败（$P_2 < B_0$或$P_2 \approx B_0$）

- 团队层（T-AVP）：

- A组：拔线3天后，团队独立完成中型功能（$P_{2,\mathrm{team}}=80$分 > $B_{0,\mathrm{team}}=68$分+$\delta$） → **T-AVP通过**
- B组：拔线3天后，团队效率显著下降（$P_{2,\mathrm{team}}=55$分 < $B_{0,\mathrm{team}}=65$分） → **T-AVP失败**

**失败模式（B组）**：

1. **能力极化**：3名资深成员能力保持，5名Junior完全依赖AI，团队整体脆弱
2. **知识流失**：团队内部不再分享经验（都问AI），隐性知识未传承
3. **架构理解差**：过度依赖AI生成代码，对系统整体理解不足，出现bug时难以定位

**定量证据**：

- A组“人际代码审查"：6个月内平均48次/人
- B组“人际代码审查"：6个月内平均12次/人
- A组Slack技术讨论：日均15条
- B组Slack技术讨论：日均5条

**可迁移洞察**：

1. **"无AI日"是简单有效的T-AVP保障机制**：成本低（只需政策）、可操作性强（每周固定1天）、副作用小（不影响整体效率）
2. **团队能力$\neq$个体能力之和**：I-AVP通过$\neq$T-AVP必然通过（涌现性）
3. **Junior成员是T-AVP的脆弱点**：最容易形成依赖，需特别保护（如前3个月禁用AI）
4. **人际交流是团队韧性的基础**：AI不能替代“隐性知识"传承，Code review、技术分享会、结对编程的价值在AI时代更加重要

**管理层决策**：基于本实验，公司决定：①全公司推广“周五无AI日"②新人前3个月禁用AI（建立基础能力）③每季度T-AVP演练（模拟AI宕机场景）④绩效考核增加“人际协作"维度。

**红旗提示**：不要将“周五无AI日"理解为“惩罚"或“开倒车"。正确定位：这是**能力锻炼日**，类似健身房的“负重训练"。如果团队抵触，可以从“每月一次"开始降低频率，或选择非关键任务进行拔线测试。

## 附录D：AVP测量协议简化版

### D.1 测量前准备清单

**适用性确认**：

- [ ] 任务类型：认知密集型、可学习的（非纯工具性任务）
- [ ] 独立完成具有价值（非补偿性外骨骼场景，见3.0.6节边界条件锚点B5）
- [ ] 可设置拔线窗口（无高风险后果）

**基线设计**：

- [ ] 设计基线任务（难度适中，完成时间30-120分钟）
- [ ] 招募参与者（最小N=30，建议$N \geq 50$）
- [ ] 记录：$B_0$分数、完成时间、主观难度（1-10量表）
- [ ] 问卷：认知负荷（NASA-TLX）、任务动机

**平行测验准备**：

- [ ] 确保任务等值性（IRT校准或专家评估）
- [ ] 准备$\geq 2$套备用任务（防止泄漏）
• 预测试验证难度一致性（pilot $N \geq 10$）
- [ ] 计算评分者间信度（ICC目标>0.7）

**参数确定**：

- [ ] $\delta$：$\geq 0.3SD$或10%（工作假设，需跨领域/任务校准）
- [ ] W窗口：4-8周（默认6周，工作假设）
- [ ] 随访时间：建议$T_2$后3个月再测（保持率评估，可选）
- [ ] 伦理审查：获得IRB/伦理委员会批准

### D.2 测量执行协议

**阶段1：基线测量（$T_0$ - 第0周）**

- 参与者在无AI辅助下完成标准任务
- 记录：原始分数、完成时间、错误类型
- 问卷：主观难度、认知负荷（NASA-TLX）、自我效能感
- 评分：由$\geq 2$名独立评分者盲评（不知道实验假设）（不知道实验假设）
- 计算评分者间信度（$ICC > 0.7$）
- 如ICC<0.7：重新培训评分者或细化评分标准
- 取平均分为$B_0$
- 质量控制：检查天花板/地板效应（如>80%或<20%达到极端分数，调整难度）

**阶段2：训练期（$T_1$→$T_2$ - 第1-8周）**

- 实验组：EML条件

- 有益摩擦：目标成功率50-70%（工作假设，需跨领域/任务校准，个体需自适应）
- 系统性削减：$S(t)$从0.8→0按曲线递减
- 每2周嵌入式微测试（10%任务无支持）

- 对照组：标准AI辅助或无AI

- 无摩擦设计（完整支持）
- 无削减机制（$S(t)$恒定）

- 记录数据：使用频率、请求帮助次数、每周任务量、周成功率、用户满意度（每2周一次）

- 执行忠实度检查：

- 确认实验组摩擦强度在50-70%（允许±5%波动）
- 确认削减曲线按计划执行
- 监测对照组是否意外引入摩擦

**阶段3：拔线窗口（W - 第9-14周，默认6周）**

- 完全停用AI辅助（技术阻断+自我报告）
- 可继续完成日常任务，但无系统支持
- 监测：
- 每周一次自我报告（是否违规使用AI）
- 行为日志抽查（如代码提交记录、写作痕迹分析）
- 违规处理：
- 轻微违规（1-2次，非关键任务）：记录但保留数据，敏感性分析
- 严重违规（$\geq 3$次或关键任务）：排除该参与者数据

**阶段4：后测（$T_3$ - 拔线窗口结束后）**

- 使用等值平行测验（与$T_0$难度相同但内容不同）
- 评分：由$\geq 2$名独立评分者盲评（不知道实验假设）
- 计算$P_2$，判定AVP结果：
- $P_2 \geq B_0 + \delta$：成功（认知内共生）
- $P_2 \approx B_0$：中性
- $P_2 < B_0$：失败（认知外骨骼）

### D.3 关键注意事项

**等值性保证**：

- $T_0$与$T_3$测验难度一致（IRT校准或专家评估）
- 内容不同（防止练习效应）
- 相同测试环境（时间、地点、指导语）

**盲评要求**：

- 评分者不知道参与者组别（实验/对照）
- 评分者不知道测试时间点（$T_0$/$T_3$）
- 评分标准提前确定并培训

**违规处理**：

- 轻微违规：保留数据，标记为“违规"，敏感性分析时排除
- 严重违规：直接排除，不纳入最终分析

**流失管理**：

- 意向性分析（ITT）：保留所有随机化参与者的数据
- 符合方案分析（PP）：仅分析完成全流程的参与者
- 如流失率>30%：分析原因，可能需要缩短W或增加激励

**伦理考虑**：

- 知情同意：参与者了解实验目的和拔线测试
- 无伤害原则：拔线测试不得用于高风险任务
- 隐私保护：AVP结果保密，不得用于雇佣/教育歧视
- 撤回权：参与者可随时退出研究

## 附录E：常见问题FAQ

**Q1：AVP是否适用于所有AI工具？**

不是。AVP仅适用于“能力增强型"人机协作，不适用于“补偿性外骨骼"（见3.0.6节边界条件锚点B5）。

**适用场景**：

- ✓ 学习工具（如编程助手、写作辅导）
- ✓ 技能训练（如设计软件、数据分析工具）
- ✓ 认知增强（如决策支持系统）

**不适用场景**：

- ✗ 残障辅助（如屏幕阅读器、义肢控制）
- ✗ 超越生理极限（如夜视设备、重力辅助外骨骼）
- ✗ 纯工具性任务（如计算器进行基础运算，无能力建构目标）

判断标准：如果工具的目标是“补偿"而非“增强"，则不适用AVP。

**Q2：如何确定$\delta$阈值？**

$\delta$（最小提升阈值）的默认工作假设为$\geq 0$.3SD或10%，但需根据领域特征进行校准：

**一般原则**：

- **统计意义**：0.3SD在心理测量学中通常被认为是“小到中等效应"，具有实践意义
- **测量误差容忍**：避免将测量噪声误判为能力提升
- **跨领域可比性**：相对阈值（SD或百分比）可适应不同任务

**领域差异**（工作假设，需实证验证）：

- **认知技能**（如编程、写作）：$\delta \geq 0.3SD$或10%
- **运动技能**（如打字速度）：可能需更大阈值（$\delta \geq 0.5SD$），因为肌肉记忆稳定性更强
- **创造性任务**（如艺术创作）：可能需定性评估而非单一$\delta$

**校准流程**：

1. 先导研究：小样本测试，确定初步参数
2. 敏感性分析：测试$\delta$变化对判定结果的影响
3. 领域专家咨询：结合实践经验调整
4. 迭代优化：根据反馈调整参数

**Q3：拔线窗口W如何选择？**

W（拔线窗口）的默认工作假设为4-8周（默认6周），但需根据任务复杂度调整：

**任务差异**（工作假设，需实证验证）：

- **快速技能**（如数学计算、简单编程）：W=4周可能足够
- **复杂技能**（如第二语言学习、高级编程）：W=8-12周
- **专业能力**（如外科手术、建筑设计）：W可能需数月甚至年

**选择依据**：

- **能力稳定性**：W需足够长，让能力从“短期记忆"转为“长期保持"
- **环境因素控制**：W不宜过长，否则混杂变量增加（如其他学习、生活变化）
- **实践可行性**：考虑参与者流失率、研究资源

**红旗**：如果W过短（<2周），可能只测到短期记忆残留，无法验证真正的能力内化。如果W过长（>12周），环境因素混杂，难以归因于AI协作。

**Q4：补偿性外骨骼为何不适用AVP？**

补偿性外骨骼（如残障辅助设备、超越生理极限的工具）与能力增强型AI的目标根本不同：

**补偿性外骨骼**：

- 目标：弥补缺失或受损的功能
- 预期：用户持续依赖工具（这是设计目标，而非缺陷）
- 评估标准：工具是否让用户达到“等效功能"（而非“能力提升"）
- 案例：盲人使用屏幕阅读器、截肢者使用义肢、老年人使用助行器

**能力增强型AI**：

- 目标：促进用户能力成长
- 预期：用户逐步独立（这是设计目标）
- 评估标准：拔线后能力是否提升（AVP）
- 案例：学习编程、提升写作、增强决策

**为什么不适用AVP**：

- 补偿性工具的“拔线测试"不合理（如要求盲人摘下屏幕阅读器阅读）
- 补偿性工具的目标不是“独立"，而是“功能等效"

**公平性原则**（见3.0.6节）：

- 对确需辅助工具的个体（如屏幕阅读器使用者），调整**任务形式**而不降低**挑战强度**
- 评估以**相对提升**而非绝对水平为准
- AVP的$\delta$阈值可个性化调整（如基于个体的$B_0$）

**Q5：如何避免Goodhart定律陷阱？**

Goodhart定律：“当一个指标成为目标时，它就不再是一个好的指标。”AVP如果被滥用为KPI，可能导致：

**风险场景**：

- 某公司将AVP用于员工晋升评估 → 员工人为操纵基线$B_0$（故意低分）、拔线期违规使用AI → AVP完全失效
- 某教育机构将AVP作为教师绩效考核 → 教师只教“容易提升"的技能，忽视重要但难以短期见效的能力

**Goodhart防护机制**（见3.0.2节注2、5.5.1节）：

1. **AVP的正确定位**：

- AVP是**验收标准**，而非**管理工具**
- AVP用于**质量判断**，而非**利益分配**

2. **分级的非KPI化**：

- AVP分级（Basic/Retention/Transfer）仅用于**质量分层**
- 禁止将AVP分数用于人事考核、绩效排名、资源分配
- 任何涉及利益分配的场景都不得使用AVP作为唯一判据

3. **固定脚注模板**（在所有阈值表格下使用）：

> *注（Goodhart防护）：本表/分级仅用于**方向与分层**；**不得下推为KPI**。最终判定以**AVP主判据（见3.0.2节）**为准。*

4. **监测与预警的分离**：

- 监测数据（如C(t)能力向量）仅用于系统改进
- 不与个人利益挂钩
- 匿名化处理，保护用户隐私

**正确使用AVP**：

- ✓ 用于评估AI工具设计的质量
- ✓ 用于研究验证理论假说
- ✓ 用于教育机构评估教学方法

**错误使用AVP**：

- ✗ 用于员工绩效考核
- ✗ 用于学生排名/分班
- ✗ 用于AI工具的营销KPI

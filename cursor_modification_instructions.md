## 附：核心改动总结

| 改动点 | 改动内容 | 改动原因 |
|-------|---------|---------|
| 30%时间分配 | 明确写出"约30%时间用于学业" | 管理候选人预期，避免误解为纯刷题 |
| 只招一个孩子 | 明确写出"不考虑弟弟" | 避免候选人误解工作量 |
| 老师收益 | 从"三行清单"升级为"三层价值+差异化" | 从"项目管理味"回到"人为什么愿意来" |
| 时间安排 | 去掉排班表，改为边界描述 | 排班表让研究型岗位像打卡工作 |
| 验收标准 | 简化，字段细节挪到入职后 | 招聘阶段不该出现入职细节 |
| 代码块规则 | 改为"禁止+允许"双向定义 | 解决四行填空不出代码块的问题 |
| 答疑模板 | 从窄型改为通用型 | 覆盖所有零散问题，不只是"能教某科目吗" |

# AI 协作模式说明

## 系统架构：三个核心组件

### GitHub 仓库 = 资料与证据库
- **作用**：存放所有计划、日志、复盘、项目材料、指标定义
- **定位**：唯一事实源（SSOT），所有决策和产出都以这里为准
- **特点**：可追踪、可复盘、可协作

### GPTS（OpenAI）= 日常教练与生成
- **作用**：把目标拆成可执行任务，生成标准化模板和文档
- **输出格式**：结论 / 下一步最小动作 / 需要输入 / 入库稿
- **使用场景**：周计划生成、任务拆解、日常问答、文档模板生成

### Claude Projects（Anthropic）= 深度研究/资料梳理
- **作用**：深度分析、方案设计、结构化复盘、研究报告
- **使用场景**：复杂问题分析、长期规划、阶段报告、深度文档修订

## 一周节奏

### 周一：周计划
- **输入**：上周复盘、INDEX 当前状态、本周目标方向
- **工具**：GPTS 生成周计划模板
- **输出**：`02_plans/YYYY-Www-plan.md`
- **更新**：同步更新 `00_admin/INDEX.md` 的"本周最小目标"

### 周二-周六：执行与记录
- **日常执行**：按计划推进任务
- **记录方式**：日打卡（可选）、实验记录、项目进展
- **工具**：GPTS 辅助生成记录模板

### 周日：周复盘
- **输入**：本周计划、执行记录、关键数据
- **工具**：Claude Projects 进行深度复盘分析
- **输出**：`03_logs/YYYY-Www-review.md`
- **更新**：同步更新 `00_admin/INDEX.md` 的"最近四周复盘"

## 两人团队模式分工

### Research Lead（研究负责人）
- **职责**：
  - 方案设计（周计划结构、实验设计）
  - 复盘分析（数据洞察、调整建议）
  - 指标体系（定义关键指标、追踪方法）
  - 阶段报告结构
  - 关键沟通（与家长对齐方向）
- **主要工具**：Claude Projects（深度分析）

### Ops Lead（执行负责人）
- **职责**：
  - 日常推进（任务执行、进度跟踪）
  - 打卡监督（运动、学习、项目进展）
  - 资料整理（归档、分类、索引）
  - 材料产出（文档、报告、演示材料）
- **主要工具**：GPTS（日常生成与模板）

## Owner/Due 规范

每个任务必须明确：
- **Owner**：谁负责执行（Research Lead / Ops Lead / 孩子）
- **Due Date**：截止日期（具体到日期，如 2025-12-25）
- **状态**：待开始 / 进行中 / 已完成 / 阻塞

示例：
```
- [ ] 生成本周周计划文档
  - Owner: Research Lead
  - Due: 2025-12-23（周一）
  - 状态: 待开始
```

## 隐私脱敏规则

### 什么必须脱敏
- 真实姓名（用代号或"孩子A"）
- 学校名称（用"某小学"或"目标学校"）
- 具体住址（用"长沙市某区"）
- 可识别照片（面部打码或使用示意图）
- 身份证号、手机号等敏感信息

### 什么不入库
- 未脱敏的个人信息
- 健康诊断记录（只记录观察和习惯建议）
- 家庭内部敏感对话
- 涉及隐私的心理评估细节

### 脱敏检查清单
- [ ] 所有对外材料（`06_reports/`）必须完全脱敏
- [ ] 项目文档中不包含可识别信息
- [ ] 周报中只使用代号或描述性语言
- [ ] 照片/视频必须打码或使用示意图

## 工作流示例

### 场景1：生成周计划
1. Research Lead 从仓库读取：上周复盘 + INDEX 当前状态
2. 粘贴到 GPTS，使用周计划模板 prompt
3. GPTS 生成：本周计划（含 Owner/Due）
4. Research Lead 检查并调整
5. 提交到仓库：`02_plans/2025-W01-plan.md`
6. 更新 INDEX.md

### 场景2：实验记录
1. Ops Lead 执行实验（如：新的运动方案）
2. 使用实验记录模板记录：目标/假设/步骤/指标/结果
3. 提交到仓库：`03_logs/2025-12-21-experiment-sport.md`
4. 周复盘时，Claude Projects 分析所有实验记录

### 场景3：周复盘
1. Research Lead 收集：本周计划 + 所有执行记录
2. 粘贴到 Claude Projects，使用周复盘模板
3. Claude Projects 生成：数据 → 洞察 → 调整
4. Research Lead 检查并补充
5. 提交到仓库：`03_logs/2025-W01-review.md`
6. 更新 INDEX.md

## 关键原则

1. **仓库是唯一事实源**：所有决策和产出必须入库
2. **AI 辅助，不替代**：AI 生成内容，人负责选择和执行
3. **可追踪、可复盘**：每个任务都有 Owner/Due，每个实验都有记录
4. **隐私优先**：默认脱敏，敏感信息不入库
5. **最小可行版本（MVP）**：先做小版本，验证后再扩展

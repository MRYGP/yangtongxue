# 周计划模板

## 文件命名
`02_plans/YYYY-Www-plan.md`（例如：`02_plans/2025-W01-plan.md`）

## 模板结构

```markdown
# 周计划 - YYYY年第W周（YYYY-MM-DD 至 YYYY-MM-DD）

## 本周 MVP（最小可行版本）
[一句话说明本周要达成的最核心目标]

## 五条主线任务

### 1. 身体发育与运动
- [ ] [任务描述]
  - Owner: [Research Lead / Ops Lead / 孩子]
  - Due: YYYY-MM-DD
  - 状态: 待开始

### 2. AI 素养与工具能力
- [ ] [任务描述]
  - Owner: [Research Lead / Ops Lead / 孩子]
  - Due: YYYY-MM-DD
  - 状态: 待开始

### 3. 学业与基础能力
- [ ] [任务描述]
  - Owner: [Research Lead / Ops Lead / 孩子]
  - Due: YYYY-MM-DD
  - 状态: 待开始

### 4. 创新与科创项目
- [ ] [任务描述]
  - Owner: [Research Lead / Ops Lead / 孩子]
  - Due: YYYY-MM-DD
  - 状态: 待开始

### 5. 表达与传播
- [ ] [任务描述]
  - Owner: [Research Lead / Ops Lead / 孩子]
  - Due: YYYY-MM-DD
  - 状态: 待开始

## 关键指标
- [指标1]: [目标值]
- [指标2]: [目标值]

## 风险与假设
- **风险1**: [描述] → 应对: [方案]
- **假设1**: [描述] → 验证方式: [方法]

## 需要输入
- [ ] [需要家长/孩子提供的信息]

## 备注
[其他说明]
```

## 示例

```markdown
# 周计划 - 2025年第1周（2025-01-06 至 2025-01-12）

## 本周 MVP（最小可行版本）
建立每日运动打卡机制，完成第一个科创项目的选题确认。

## 五条主线任务

### 1. 身体发育与运动
- [ ] 设计每日运动打卡表（跳绳500个 + 拉伸10分钟）
  - Owner: Ops Lead
  - Due: 2025-01-07
  - 状态: 待开始
- [ ] 监督并记录本周运动数据
  - Owner: Ops Lead
  - Due: 2025-01-12
  - 状态: 待开始

### 2. AI 素养与工具能力
- [ ] 教孩子使用 ChatGPT 辅助学习（错题分析）
  - Owner: Research Lead
  - Due: 2025-01-08
  - 状态: 待开始

### 3. 学业与基础能力
- [ ] 用 AI 工具分析本周错题，生成错题本
  - Owner: 孩子（Ops Lead 协助）
  - Due: 2025-01-10
  - 状态: 待开始

### 4. 创新与科创项目
- [ ] 完成项目选题确认（3个候选方案 → 1个确定）
  - Owner: Research Lead + 孩子
  - Due: 2025-01-09
  - 状态: 待开始

### 5. 表达与传播
- [ ] 让孩子对 AI 讲解本周学习内容（5分钟）
  - Owner: 孩子（Research Lead 协助）
  - Due: 2025-01-11
  - 状态: 待开始

## 关键指标
- 运动打卡完成率: ≥ 5/7 天
- 项目选题确认: 1个确定方案
- AI 工具使用: 至少使用 ChatGPT 分析错题1次

## 风险与假设
- **风险1**: 孩子可能对打卡有抵触 → 应对: 降低初始目标，先建立习惯
- **假设1**: 孩子对选题有明确偏好 → 验证方式: 让孩子列出3个想法，讨论筛选

## 需要输入
- [ ] 家长确认本周时间安排
- [ ] 孩子提供3个项目想法

## 备注
本周重点是建立基础机制，不求完美，先跑通流程。
```

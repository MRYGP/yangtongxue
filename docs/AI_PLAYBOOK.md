# AI 协作手册（Claude Projects / GPTS / Cursor）

## 三者协同

### Claude Projects（Anthropic）
- **角色**: 深度研究/资料梳理
- **使用场景**: 复杂问题分析、长期规划、阶段报告、深度文档修订
- **输入**: 仓库内容（SYNC、Plan、Retro）
- **输出**: 深度分析报告、结构化复盘

### GPTS（OpenAI）
- **角色**: 日常教练与生成
- **使用场景**: 周计划生成、任务拆解、日常问答、文档模板生成
- **输入**: SYNC、Plan、用户指令
- **输出**: 标准化文档（四段式结构）

### Cursor（本地 IDE）
- **角色**: 代码编辑与文件管理
- **使用场景**: 直接编辑仓库文件、代码生成、文件操作
- **输入**: 用户指令、仓库上下文
- **输出**: 代码修改、文件创建/更新

## 协作流程

### 场景1: 生成周计划
1. **Claude Projects** 读取 SYNC 和上周复盘，进行深度分析
2. **GPTS** 基于分析结果生成周计划模板
3. **Cursor** 创建/更新 `02_plans/YYYY-Www-plan.md`
4. 人工检查并提交到仓库

### 场景2: 周复盘
1. **GPTS** 收集本周数据和执行记录
2. **Claude Projects** 进行深度复盘分析（数据→洞察→调整）
3. **Cursor** 创建/更新 `03_logs/YYYY-Www-review.md`
4. 更新 SYNC.md 和 INDEX.md

### 场景3: 项目推进
1. **GPTS** 生成任务拆解和模板
2. **Cursor** 更新项目文档
3. **Claude Projects** 进行阶段性复盘和调整建议

## 输出协议（统一）

无论使用哪个 AI 工具，输出都应包含:

1. **结论**: 核心结论/产出
2. **下一步最小动作**: 具体可执行的下一步（含 Owner 和 Due）
3. **需要输入**: 需要人工提供的信息/决策
4. **入库稿**: 可直接提交到仓库的文档/代码片段

## 关键原则

1. **仓库是唯一事实源**: 所有 AI 输出最终沉淀到仓库
2. **工具互补**: 不同工具用于不同场景，不重复工作
3. **格式统一**: 所有输出遵循统一格式规范
4. **隐私优先**: 所有生成内容必须脱敏
